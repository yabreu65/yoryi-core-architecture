# PR Architecture Checklist

Checklist obligatorio para PRs con impacto de arquitectura.

- [ ] El PR referencia docs de `constitution/` y `domains/` aplicables.
- [ ] El cambio respeta limites de modulo y contratos.
- [ ] Se evaluo impacto multi-tenant y riesgo de leakage.
- [ ] Se actualizaron docs/checklists/templates si cambio una regla.
- [ ] Se creo ADR si hubo decision D2 o D3.
- [ ] Se incluyen evidencias de testing y observabilidad minima.
