# Versioning Policy

## Semantic versioning documental

- MAJOR: cambia principios o rompe compatibilidad de procesos.
- MINOR: agrega doctrina/patrones/checklists sin romper flujo.
- PATCH: correcciones editoriales o clarificaciones.

## Release cadence

- Release mensual para MINOR/PATCH.
- MAJOR por ventana aprobada por arquitectura.

## Compatibility contract

`AGENTS.md` y `catalog/knowledge-map.md` deben mantener estabilidad entre releases para no romper agentes.
