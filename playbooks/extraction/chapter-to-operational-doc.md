# Chapter to Operational Doc

Metodo para transformar contenido pedagogico en conocimiento accionable.

## Paso 1 - Segmentacion

Dividir capitulo por decisiones, no por secciones narrativas.

## Paso 2 - Normalizacion

Para cada decision crear doc con formato fijo:

- Contexto
- Regla
- Cuando aplica / no aplica
- Trade-offs
- Checklist
- Referencias

## Paso 3 - Atomicidad

Un problema por archivo. Si el capitulo mezcla 3 problemas, crear 3 docs.

## Paso 4 - Trazabilidad

Agregar `source_refs` al frontmatter y actualizar `catalog/source-traceability.md`.

## Paso 5 - Enforcement

Toda regla nueva debe tener checklist o quality gate asociado.
