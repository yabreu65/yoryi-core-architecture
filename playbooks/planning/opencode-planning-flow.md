# OpenCode Planning Flow

Flujo operacional para planificacion con OpenCode.

## Step 1 - Context loading

Leer siempre:

- `catalog/knowledge-map.md`
- `constitution/architecture-principles.md`
- `constitution/decision-policy.md`

Luego leer docs del dominio impactado.

## Step 2 - Decision framing

Definir:

- objetivo tecnico,
- restricciones,
- criterios de exito,
- riesgos.

## Step 3 - Architecture validation

Aplicar checklists en `checks/` y registrar gaps.

## Step 4 - ADR gate

Si hay cambio estructural o excepcion, generar ADR.

## Step 5 - Plan output

Entregar plan con:

- contexto consultado,
- decisiones y trade-offs,
- evidencia de checklist,
- riesgos y mitigaciones,
- ADR o justificacion.
